Projects

Product Sales Analysis

Developed a classification model to predict product sales conversion status (e.g., 'CONVERTED' vs. 'LOST') using a dataset of 7,420 sales records retrieved from a MySQL database. Conducted robust data preprocessing, including cleaning duplicate entries, feature engineering (extracting temporal features like 'Duration' and 'Hour'), and employing the SMOTE technique to successfully handle class imbalance in the target variable. Evaluated a suite of machine learning models (Logistic Regression, SVM, Neural Network, Decision Tree, and XGBClassifier), ultimately achieving a best-in-class accuracy of 0.69 using the XGBoost Classifier. The final optimized model was prepared for deployment by saving it as a pickle file, enabling its use in a production environment to aid sales strategy.

Twitter Sentiment Analysis

Large-Scale Data Handling: Successfully loaded, cleaned, and managed a large-scale Twitter dataset containing over 1 million rows and multiple features, demonstrating proficiency with pandas for big data manipulation. Data Quality Assessment: Performed comprehensive data checks (including .head(), .info(), .describe(), and .isnull().sum()) to confirm dataset integrity, ensure no missing values, and understand column data types. Feature Engineering & Cleaning: Identified and dropped irrelevant features (id of the tweet, date of the tweet, query) and proactively renamed ambiguous columns (polarity of tweet to label, text of the tweet to text) for clarity and model compatibility. Target Variable Transformation: Handled non-standard class labels by transforming the sentiment polarity from the original [0, 4] (Negative/Positive) to a standard binary classification label [0, 1], ensuring a consistent target variable for the model.
